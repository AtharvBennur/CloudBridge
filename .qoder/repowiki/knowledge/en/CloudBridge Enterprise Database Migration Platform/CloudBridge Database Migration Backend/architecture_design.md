The `app/__init__.py` `create_app()` factory is the single wiring point: it loads environment-driven config via `app.config`, initializes shared extensions (`db`, `cors`, `socketio`) from `app.extensions`, registers cross-cutting logging and error handlers, then imports and registers every domain blueprint (auth, aws_connection, database_config, migration/migration_engine, cdc, schema_drift/schema_approval, ecs, notification, rollback, health, preflight, observability) plus Socket.IO event handlers in `routes/websocket.py`. All domains share one SQLAlchemy metadata (`app.models.*`) and one Pydantic-style schema layer (`app.schemas.*`). Long-running work (CDC polling, ECS task watching) runs through the worker pool in `app.workers.manager` using a base class in `base_worker.py`. The entrypoint `run.py` boots the app and enforces SECRET_KEY outside debug mode; Docker builds via `Dockerfile` with an isolated `venv/`.
