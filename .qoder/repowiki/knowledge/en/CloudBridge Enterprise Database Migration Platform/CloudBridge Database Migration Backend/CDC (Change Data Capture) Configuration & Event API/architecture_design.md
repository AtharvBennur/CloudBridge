Three-layer Flask app slice under `app/`:
- `routes/cdc.py` defines a single Blueprint (`/cdc`) with `@login_required` guards; it owns HTTP I/O, request parsing, error handlers for `CDCValidationError`, `CDCConfigNotFoundError`, `CDCServiceError`, and delegates all business logic to `CDCService`. It also directly drives the in-process worker lifecycle via `worker_manager` (start/pause/resume/stop) and persists status transitions on `MigrationJob`.
- `services/cdc_service.py` is stateless except for an optional injected logger; it performs DB reads/writes through `app.extensions.db`, enforces referential integrity against `MigrationJob`, and exposes methods for config CRUD, event recording (`record_change_event`), event lifecycle (`mark_event_processed`, `mark_event_failed`), lag metrics, and statistics aggregation.
- `schemas/cdc.py` holds frozen `dataclass` DTOs (`CreateCDCConfigRequest`, `UpdateCDCConfigRequest`, `CDCConfigResponse`, `CDCEventResponse`) with `from_payload` / `from_model` / `to_dict` converters that validate enum membership against `CDCMode.VALUES` / `CDCStatus.VALUES` and coerce integers.
- `models/cdc_config.py` and `models/cdc_event.py` define the two SQLAlchemy tables (`cdc_configs`, `cdc_events`) plus string-based enum classes (`CDCMode`, `CDCStatus`, `ChangeOperation`, `CDCEventStatus`) whose `.VALUES` sets drive schema validation. Both models use `db.Column(..., server_default=text("CURRENT_TIMESTAMP"))` timestamps and index `migration_id`, `table_name`, and `lsn`.
Dependency direction: routes → service → models; schemas depend only on models for type hints. The route layer has a one-way coupling into `app.workers.manager` and `app.workers.cdc_worker.PostgreSQLCDCWorker` to start/stop replication workers per migration job.
