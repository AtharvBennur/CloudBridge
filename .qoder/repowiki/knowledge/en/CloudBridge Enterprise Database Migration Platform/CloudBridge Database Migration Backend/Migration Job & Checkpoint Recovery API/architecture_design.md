Two parallel Flask blueprints under `app/routes/` expose the API surface: `migration.py` (`/migrations`) provides full CRUD over `MigrationJob`, while `rollback.py` (`/rollback`) exposes checkpoint lifecycle endpoints (`/checkpoint`, `/to-checkpoint/<id>`, `/resume/<id>`, `/restart/<id>`, `/checkpoints/<id>`, `/recovery-options/<id>`, `/cleanup-checkpoints/<id>`). Each blueprint instantiates a single service object (`MigrationService`, `RollbackService`) and delegates all business logic there; routes stay thin — they only parse JSON, call the service, and jsonify responses. The service layer owns validation (via frozen dataclass schemas in `schemas/migration.py`), persistence through `app.extensions.db.session`, and structured logging via `current_app.logger`. Models are plain SQLAlchemy classes in `models/migration.py` (`MigrationJob`, `MigrationStatus` enum) and `models/migration_checkpoint.py` (`MigrationCheckpoint`, foreign-keyed to `migration_jobs`). Cross-cutting concerns: `@login_required` guards every endpoint; each blueprint registers its own `errorhandler`s mapping domain exceptions (`MigrationError`, `MigrationNotFoundError`, `MigrationValidationError`; `RollbackServiceError`, `CheckpointNotFoundError`, `RollbackValidationError`) to `{"error":{"message":...}}` JSON with 400/404 status codes. Dependency direction is strictly Routes → Services → Schemas + Models; services also reach into sibling models (`CDCConfig`, `CDCEvent`) when resetting CDC state during rollback/restart.
