Three-layer Flask app organized by feature:
- `routes/` — two independent blueprints (`schema_approval`, `schema_drift`) under `/schema-approval` and `/schema-drift`; each blueprint registers its own `@errorhandler` for domain exceptions and delegates all work to a service class.
- `services/` — `SchemaDriftService` owns snapshot capture (via simulated `_extract_*` helpers), diffing logic (`_detect_table_changes`, `_detect_column_changes`, `_detect_index_changes`), and event lifecycle (approve/reject/ignore); `SchemaApprovalService` composes it to coordinate with the migration runner (`MigrationJob`), pausing/resuming jobs based on `SchemaChangeRisk` thresholds.
- `schemas/schema_drift.py` — frozen dataclass DTOs with `from_payload` / `from_model` + `to_dict` converters enforce request validation and serialize responses; no Pydantic is used.
- `models/schema_snapshot.py` — SQLAlchemy models `SchemaSnapshot` and `SchemaDriftEvent` plus enum-like classes `SchemaChangeType` / `SchemaChangeRisk` holding string constants and a `VALUES` set. JSON blobs are persisted as `db.Text` columns and round-tripped via `json.dumps`/`json.loads` in services and DTOs.
Dependency direction: routes → services → models/extensions; `SchemaApprovalService` depends on `SchemaDriftService`, never the reverse. Auth is enforced uniformly via the shared `app.middleware.auth.login_required` decorator.
