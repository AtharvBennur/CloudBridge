The repo is a two-service monorepo orchestrated by `docker-compose.yml`: a `db` (PostgreSQL 16) service shared by both children, a `backend` (Flask + SQLAlchemy + Socket.IO) service that owns all persistence and AWS integration, and a `frontend` (React/Vite SPA) built via multi-stage Dockerfile and served on port 80. The compose file enforces startup order (`db` healthy → `backend` healthy → `frontend`) and bridges them over a single `cloudbridge` bridge network. The frontend is configured at build time through `VITE_API_BASE_URL` / `VITE_WS_BASE_URL` args so it targets the backend's `/health`, REST routes, and Socket.IO endpoints; the backend reads its own `DATABASE_URL` and `.env` from `./backend/.env`. There is no cross-child source code — communication is strictly HTTP/REST + WebSocket over the compose network, with the root level only providing the runtime wiring and environment templates (`.env.example`).
